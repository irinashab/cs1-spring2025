Lesson 3: More Interactions

Interaction with the Mouse

Java (and subsequently Processing) has event listeners that listen for messages from the operating system, such as:

  • “The mouse moved within the canvas.”
  • “The mouse was clicked at this point on the canvas.”

When such an event occurs, the program performs an action and then waits for more messages.

Built-in Mouse Variables

To enable interactivity, we need setup() and draw() functions. Processing provides built-in keywords:

  • mouseX - stores the x-coordinate of the mouse.
  • mouseY - stores the y-coordinate of the mouse.

Example: Following the Mouse

void setup() { 
  size(480, 270); 
  noStroke(); 
} 

void draw() { 
  background(255); 
  fill(29, 215, 112); // Shade of green
  circle(mouseX, mouseY, 50); 
}

The circle follows the mouse!

Try This:

  1. Move background() from draw() to setup() and observe the difference.
    • Can you explain what happens?
  2. Return background() to draw(), then use mouseX and mouseY as fill colors.
    • Why is the initial color dark? Why does it lighten when moving the mouse to the lower-right corner?

Creating More Interesting Movements

Instead of having the circle exactly follow the mouse, we can modify its position:

void setup() { 
  size(500, 500); 
  noStroke(); 
  noCursor(); // Hides the cursor 
  fill(232, 23, 200); // Pinkish color
} 

void draw() { 
  background(126); // Shade of gray
  ellipse(mouseX, mouseY, 50, 50);    // Top circle
  ellipse(mouseX + 30, mouseY + 45, 50, 50); // Middle circle
  ellipse(mouseX - 30, mouseY + 80, 50, 50); // Bottom circle
}

Observation: The three circles stay at fixed distances from each other!

Example: Scaled Offsets

void setup() { 
  size(500, 500); 
  noStroke(); 
  noCursor(); 
  fill(232, 23, 200); 
} 

void draw() { 
  background(126); 
  ellipse(mouseX, mouseY, 50, 50);    // Top circle
  ellipse(mouseX / 2, mouseY + 45, 50, 50); // Middle circle
  ellipse(2 * mouseX, mouseY + 80, 50, 50); // Bottom circle
}

Multiplying or dividing mouseX and mouseY creates scaled offsets.

Previous Mouse Position

Processing provides:

  • pmouseX - stores the previous x-coordinate of the mouse.
  • pmouseY - stores the previous y-coordinate of the mouse.

If the mouse is still, these values are the same as mouseX and mouseY. If the mouse moves quickly, the difference increases.

Example: Observing pmouseX & mouseX

void setup() { 
  size(500, 500); 
}

void draw() { 
  frameRate(12); 
  println(pmouseX - mouseX); 
}

Move the mouse slowly and quickly to see the changing values in the console.

Example: Drawing a Trail

void setup() { 
  size(480, 270); 
  background(255); 
  strokeWeight(5); // Line width
  stroke(237, 123, 42); // Line color
}

void draw() { 
  line(mouseX, mouseY, pmouseX, pmouseY); 
}

Mouse Clicks & Key Presses

Processing can also respond to mouse clicks and key presses with:

void mousePressed() { 
  // Code for mouse click event
} 

void keyPressed() { 
  // Code for key press event
}

Example: Interactive Drawing

void setup() { 
  size(480, 270); 
  background(255); 
  noStroke(); 
  fill(40, 250, 122); 
} 

void draw() {}

void mousePressed() { 
  circle(mouseX, mouseY, 16); 
} 

void keyPressed() { 
  background(255); // Clears the screen
}

Explanation:

  • Clicking the mouse draws a small circle where the cursor is.
  • Pressing any key resets the background, erasing all circles.

Keep experimenting to create more dynamic visuals!
